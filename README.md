# 基于《千恋＊万花》角色设定的 AI 小镇恋语 RPG 原型项目
Developed with Unreal Engine 5

> 一个以 **Unreal Engine 5.7** 为核心、围绕 **AI 驱动 NPC 交互 + 小镇日常模拟 + 战斗成长闭环** 构建的二创原型项目。
> 本项目重点展示了我在 **游戏客户端架构设计、核心系统开发、LLM 交互链路接入、状态持久化与多模块联调** 方面的完整实践能力。

---

## 项目简介

本项目以《千恋＊万花》的角色设定为灵感，构建了一个可运行的 **AI 小镇 RPG 原型**。玩家可以在小镇中探索、与特殊 NPC 对话、提升关系、参与战斗、获取掉落、管理背包与商店，并通过存档系统保持关键状态一致性。

项目的目标并不是简单复刻原作内容，而是围绕“**如何将大模型能力自然地嵌入 RPG 交互体验**”这一方向，完成一个具备较完整客户端系统的原型验证。

---

## 项目亮点

### 1. AI 驱动的角色对话交互

* 接入本地 **LM Studio（OpenAI Compatible API）**
* 构建面向 NPC 的 **LLM 交互链路**
* 支持 **Prompt Engineering、角色条件控制、会话历史管理、上下文裁剪、结构化响应解析**
* 支持将模型输出映射到 **好感度变化与游戏数值系统**

### 2. 完整的 RPG 客户端基础框架

* 基于 **UE5.7 + C++ + Blueprint** 搭建规范化 Gameplay 架构
* 完成 **角色控制、交互系统、UI 系统、任务系统、时间/天气联动、战斗系统、敌人 AI** 等核心模块开发
* 具备较完整的“**日常交互—战斗成长—资源循环**”玩法闭环

### 3. 多系统联动与状态持久化

* 实现 **存档、背包、掉落、商店、任务进度、NPC 好感度、战斗属性** 等系统
* 支持 **玩家位置、时间、背包、金钱、任务进度、商店库存、角色属性** 等关键状态恢复
* 重点解决多模块联动下的读档一致性问题

### 4. 以 AI 作为研发提效工具，而非完全依赖

在项目开发过程中，结合代码大模型辅助进行：

* 样板代码生成
* Prompt 迭代与对话调优
* 问题定位与调试辅助
* 方案验证与快速原型实现

核心架构设计、系统拆分、关键逻辑实现与联调验证均由我独立完成。

## 核心功能一览

### 1. 角色控制与交互系统

* 第三人称角色移动、镜头控制
* 基于统一交互逻辑的 NPC / 掉落物 / 商店对象交互
* 交互提示 UI 与输入模式切换

### 2. AI NPC 对话系统

* 特殊 NPC 使用独立角色设定与 Prompt 模板
* 支持上下文历史拼接与会话连续性
* 支持从模型回复中解析结构化字段
* 支持将对话结果映射为好感度变化

### 3. 好感度与属性联动

* 特殊 NPC 的关系变化可影响主角属性
* 属性联动结果直接写入角色战斗数据
* 关系、属性、战斗系统形成正反馈闭环

### 4. 战斗与敌人 AI

* 敌人自动生成与区域约束
* 基于导航与行为树的敌人 AI 行为
* 掉落系统、击杀统计、任务推进
* 战斗属性与角色成长联动

### 5. 时间、天气与小镇运行逻辑

* 自定义游戏时间推进规则
* 保存后时间跳转与读档恢复
* 天空/天气系统与时间同步
* NPC 日程刷新与怪物定时刷新

### 6. 背包、掉落与商店系统

* 基于 Data Asset 的物品配置
* 地上掉落拾取与资源入包
* 文字版背包 UI、道具使用
* 商店购买、库存变化与状态持久化

### 7. 存档系统

* 保存并恢复：

  * 玩家位置
  * 当前时间
  * 战斗属性
  * 背包与金钱
  * 任务进度
  * NPC 好感度
  * 商店库存
* 重点保证多模块状态恢复的一致性

## 关键技术设计

### 一、Gameplay 架构

项目基于 UE 常规 Gameplay 体系组织逻辑：

* **GameMode**：规则入口与局部流程管理
* **GameState**：时间、全局状态与部分广播
* **PlayerController**：UI、输入模式与菜单管理
* **Character**：角色状态、任务进度、交互桥接
* **Actor / Component**：敌人、掉落物、背包、商店等功能模块拆分

这种设计方式使得后续系统扩展时，能够较清晰地划分“输入层、表现层、状态层、规则层”的职责。

---

### 二、LLM 交互链路

我在项目中将大模型能力封装为一条可控的交互链路，而不是简单调用聊天接口。

主要包括：

* 角色 Prompt 模板
* 角色条件信息注入
* 对话历史管理
* 响应清洗与解析
* 数值反馈映射

这样做的目的是让模型输出不仅“像在说话”，还能够服务于游戏系统本身。

---

### 三、状态持久化设计

项目在后期的重点之一，是保证多模块并行开发后仍能实现稳定读档。

当前存档系统已覆盖：

* 时间
* 战斗属性
* 背包与金钱
* 任务进度
* NPC 好感度
* 商店库存

相比只做位置与时间恢复，这一部分更偏向客户端系统整合与工程稳定性验证。

## 运行说明

### 开发环境

* Unreal Engine 5.7
* Visual Studio 2026
* Windows 11

### 启动项目

1. 克隆仓库并拉取 LFS 文件
2. 使用 UE5.7 打开 `.uproject`
3. 根据本地环境配置必要插件与依赖
4. 编译 C++ 代码后运行项目

### LLM 相关

若需要启用 NPC 大模型对话：

1. 启动本地 **LM Studio**
2. 加载可用模型
3. 开启 OpenAI 兼容服务
4. 在项目配置中填写对应地址与参数
